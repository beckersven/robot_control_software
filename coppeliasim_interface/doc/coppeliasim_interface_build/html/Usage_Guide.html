Usage Guide¶

Start-Up¶

To launch the CoppeliaSim-Interface, one has to do following steps in correct order. In the beginning, no ROS or CoppeliaSim should be running. Be sure to always source required bash-files!

  1. Open 3 terminal windows
  2. In the first terminal, start the ROS-framework with roscore
  3. In the second one, navigate via cd to CoppeliaSim and launch it: ./coppeliaSim.sh
  4. Wait until CoppeliaSim is launched, then open and configure the basic_scene.ttt (see the separate documentation)
  5. In the third terminal, run: roslaunch coppeliasim_interface simulation_bringup.launch
  6. Wait until MoveIt’s GUI is fully loaded and perform basic planning-/execution-tasks to ensure functionality

Scanner¶

  • Output
    • /current_scan_line (sensor_msgs/PointCloud): What the Laser-Scanner measures at the moment
    • /stitched_point_cloud (sensor_msgs/PointCloud): Combined point cloud of all measurements while stitching was active
    • Visualization in RVIZ
      1. At the bottom of the left panel “Displays”, click “Add”
      2. Switch the right tab and add the above topic(s)
      3. Optional: In the left panel “Displays”, customize the point cloud’s appearance (you might need to scroll down to do so)
    • Color matches sensor precision: Green = low uncertainty, red = high uncertainty, gradient inbetween
  • Stitch-Control
    • Service /clear_stitched_point_cloud (std_srvs/Trigger): Clear the stitched point cloud
    • Service /switch_stitch_mode (std_srvs/SetBool): Enables stitching, if request’s data is set true, or disables it otherwise
    • Stitching is disabled on start-up
